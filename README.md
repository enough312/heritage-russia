# Культурное наследие народов России

Статический многостраничный конкурсный сайт на чистых `HTML`, `CSS` и `JavaScript`.

## Что внутри

- `index.html` — главная страница
- `peoples.html` — народы России
- `traditions.html` — традиции и праздники
- `crafts.html` — ремесла и архитектура
- `cuisine.html` — кухня народов России
- `music.html` — музыка и эпос
- `routes.html` — культурные маршруты
- `preservation.html` — сохранение наследия
- `404.html` — страница ошибки для публикации
- `styles.css` — общий дизайн и адаптив
- `content.js` — все данные сайта
- `app.js` — интерактивность, галереи, фильтры, lightbox, шапка
- `assets/` — декоративные SVG-элементы

## Как открыть локально

Можно просто открыть `index.html` в браузере.

Для более стабильной проверки лучше запустить локальный сервер:

```powershell
py -m http.server 4173
```

После этого открыть:

```text
http://localhost:4173
```

## Проект готов к показу

- все основные страницы статические и работают без сборки
- медиафайлы подхватываются из корня проекта по названиям
- контакты автора уже указаны в `content.js`
- для GitHub Pages добавлен файл `.nojekyll`

## Быстрая публикация на GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Загрузи в него все файлы из этой папки, чтобы `index.html` лежал в корне репозитория.
3. Открой репозиторий на GitHub.
4. Перейди в `Settings` → `Pages`.
5. В блоке `Build and deployment` выбери:
   - `Source` → `Deploy from a branch`
   - `Branch` → `main`
   - `Folder` → `/ (root)`
6. Нажми `Save`.
7. Подожди 1-3 минуты и обнови страницу `Pages`.
8. GitHub покажет готовую ссылку на сайт.

Если репозиторий называется, например, `heritage-site`, то ссылка обычно будет такой:

```text
https://ТВОЙ-ЛОГИН.github.io/heritage-site/
```

## Самый простой вариант без GitHub

Можно опубликовать сайт через Netlify Drop:

1. Открой `https://app.netlify.com/drop`
2. Перетащи в окно всю папку проекта
3. Дождись загрузки
4. Получи готовую ссылку сразу после публикации

## Что важно перед загрузкой

- не менять имена HTML-файлов без необходимости
- не убирать `index.html` из корня
- загружать вместе HTML, CSS, JS, `assets/` и все изображения/видео
- если добавляешь новые медиафайлы, клади их рядом с проектом и называй максимально близко к названиям карточек
